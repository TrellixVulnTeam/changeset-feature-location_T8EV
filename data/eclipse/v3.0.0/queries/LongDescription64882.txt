I200405290105 (3.0RC1) If an exception occurs in an inner class the stack-trace may be printed to theconsole. In 3.0M7 you could click on a message like  "java.lang.NullPointerException	at Test$1.doIt(Test.java:9)" and the editor would be positioned to line 9 in Test.java. In 3.0RC1 this does no longer work. Example:  public class Test {    interface IFoo {        void doIt();    }     public static void main(String[] args) {        IFoo foo = new IFoo() {            public void doIt() {                String s = null;                int i = s.length();            }        };        foo.doIt();    }} In 3.0M7 you can click on "at Test$1.doIt(Test.java:10)" and the editor getspositioned correctly. in 3.0RC1 you only get a dialog telling you that "Source not found for Test$1".
