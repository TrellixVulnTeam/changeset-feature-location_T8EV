\begin{abstract}
Feature location is a program comprehension activity in which a developer
inspects source code to locate where features are implemented.
Feature location technique research traditionally build topic models on source code snapshots.
However, source code evolves, which causes models to become obsolete,
leading to retraining the source code topic models.
In this paper, we introduce an approach to topic modeling \emph{changesets} for feature location.
This approach incrementally updates the model, avoiding the computational cost of keeping it up-to-date.
We conducted an empirical study in to evalute changeset-based feature location.
We demonstrate the robustness of this approach by using two popular information retrieval models --
latent semantic indexing and latent Dirichlet allocation.
Overall, we studied over 1200 bugs and features from fourteen open source Java projects.
Additionally, we perform a study on feature location in a \emph{temporal simulation}.
That is, we approximate the performance of our approach as if it were used in real developement.
Our findings suggest that our approach can perform on par with source code-based approaches,
but may not be the best approach for newly formed projects.
\end{abstract}
